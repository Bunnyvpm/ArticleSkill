# 文章创作工作流程

> 综合使用所有Skill，创作高质量爆款文章

## 🎯 核心原则

当用户提出新文章需求时，AI助手必须：

1. **同时参考所有Skill文件** - 不能只看单一文件
2. **学习用户写作风格** - 从作者配置.md和现有文章中提取风格特征
3. **抓取相关热点内容** - 确保话题具有时效性和热度
4. **保持人设一致性** - 严格按照作者配置.md的人设写作
5. **提供可落地方法** - 每篇文章都要有具体可操作的内容

**重要**：如果用户未填写作者配置.md，AI应首先引导用户填写或询问核心人设特征。

## 🔄 完整创作流程

### 阶段1：需求理解与资源加载（第一步）

**用户输入**：
```
"我要写一篇关于[主题]的文章"
或
"帮我写[类型]的文章，关于[内容]"
```

**AI执行步骤**：

```markdown
## Step 1: 确认需求
- 明确文章类型：工具介绍 / 方法教程 / 深度思考 / 其他
- 明确核心主题：要写什么
- 明确特殊要求：篇幅、侧重点、发布时间等

## Step 2: 检查作者配置
检查是否存在作者配置.md：
- 如果存在 → 读取并提取人设特征
- 如果不存在 → 引导用户填写，或询问核心人设问题：
  * 你的笔名是什么？
  * 你的身份和领域是什么？
  * 你希望文章风格是（亲和力强/专业严谨/幽默风趣/极简干货）？
  * 举一个你觉得能代表你风格的例子

## Step 3: 加载所有Skill文件
使用Read工具读取：
- skills/公众号爆款写作.md
- skills/深度认知写作.md
- skills/热点分析工具.md
- skills/写作辅助工具.md
- skills/去AI化润色.md
- skills/快速参考.md
- skills/作者配置.md（如果存在）

## Step 4: 学习用户风格
- 找到与当前类型最相似的文章（用户指定或自动检索）
- 提取风格特征：
  * 开头钩子方式
  * 段落长度习惯
  * 情绪表达方式
  * 金句位置和频率
  * 互动引导方式
  * 专业术语使用频率

## Step 5: 抓取热点内容（如需要）
- 使用WebSearch工具搜索相关热点
- 分析热点话题的切入角度
- 提取可借鉴的爆款要素
```

**输出确认**：
```
✅ 已加载写作方法论
✅ 已读取作者配置：[人设标签]
✅ 已学习你的风格特征
✅ 已抓取[主题]相关热点
✅ 准备开始创作
```

---

### 阶段2：选题与标题（第二步）

**AI执行步骤**：

```markdown
## Step 1: 话题分析
- 基于热点内容，分析切入角度
- 结合用户定位，确定差异化观点
- 评估话题的爆款潜力

## Step 2: 标题生成
- 使用4种标题公式各生成3-5个选项
- 标注每个标题的亮点分析
- 推荐最符合作者人设的标题

输出格式：
【标题选项】
类型：冲突对比型
标题：[标题文字]
亮点：[分析为什么好]
推荐指数：⭐⭐⭐⭐⭐
```

**用户确认**：
```
用户：选择第2个标题
或
用户：融合一下1和3
或
用户：都不满意，重新生成
```

---

### 阶段3：结构搭建（第三步）

**AI执行步骤**：

```markdown
## Step 1: 选择结构模板
- 根据文章类型选择对应模板
- 工具介绍型 / 方法教程型 / 深度思考型
- 或参考深度认知写作.md的六部分框架

## Step 2: 生成详细大纲
- 按模板生成完整大纲
- 标注每部分字数
- 标注配图位置和类型
- 标注金句插入位置
- 标注互动引导位置

## Step 3: 风格适配
- 参考学习到的风格特征
- 参考作者配置.md中的人设
- 调整大纲细节
- 确保符合作者风格

输出格式：
【文章大纲】
类型：[文章类型]
预计字数：[总字数]
风格：[基于作者配置的风格标签]

一、[标题1]（字数）
  - 要点1
  - 要点2
  配图建议：[配图类型]

二、[标题2]（字数）
  ...
```

**用户确认**：
```
用户：大纲OK，开始写
或
用户：调整一下第3部分
或
用户：我想多加一个XX部分
```

---

### 阶段4：内容创作（第四步）

**AI执行步骤**：

```markdown
## Step 1: 保持人设写作
✅ 第一人称"我"（如果适合该风格）
✅ 真实感：分享踩坑经历
✅ 情绪化：符合作者的情绪表达习惯
✅ 自嘲适度：根据作者风格调整
✅ 赋能读者："连我也能做到"

## Step 2: 遵循模板结构
- 严格按照确定的大纲写作
- 控制每部分字数
- 在标注位置插入配图提示

## Step 3: 学习风格特征
- 参考现有文章的：
  * 开头钩子写法
  * 段落长度（≤5行）
  * 过渡句方式
  * 专业术语使用频率

## Step 4: 添加爆款要素
- 在适当位置插入金句（标注【金句】）
- 设计互动提问（标注【互动】）
- 标注配图位置（标注【配图】）
- 设计情绪高潮点

## Step 5: 减少条目式输出
- 优先使用流畅段落式表达
- 避免过度使用列表和编号
- 用逻辑递进替代并列罗列

输出格式：
【文章正文】

[开头钩子]
内容...

【配图：图片类型描述】

[痛点场景]
内容...

【金句】金句内容【金句】
...
```

**用户反馈**：
```
用户：这段写得好
用户：那段太干了，改一下
用户：加个具体的例子
```

---

### 阶段5：优化与检查（第五步）

**AI执行步骤**：

```markdown
## Step 1: 内容优化
- 检查语言风格是否符合人设（作者配置.md）
- 检查段落长度是否合适
- 检查情绪表达是否到位
- 优化不够吸引力的部分

## Step 2: 去AI化处理
- 去除机械连接词
- 去除AI高频词汇
- 增加人性化细节
- 调整节奏和韵律

## Step 3: 爆款要素检查
使用检查清单逐项检查：
□ 标题有冲突/对比/意外
□ 开头3秒抓住注意力
□ 每300字一张图
□ 每500-800字一个小标题
□ 有情绪化表达（根据风格）
□ 有金句输出
□ 有互动引导
□ 有行动号召
□ 解决真实痛点
□ 有可落地方法

## Step 4: 风格一致性检查
- 与学习到的文章风格对比
- 与作者配置.md对比
- 检查是否有遗漏的要素
- 确保符合作者人设

## Step 5: 最终调整
- 根据检查结果调整
- 标注需要用户补充的部分（如配图）
- 给出发布建议

输出格式：
【优化完成】

✅ 已完成内容优化
✅ 已完成去AI化处理
✅ 已通过爆款要素检查
✅ 已符合作者风格

【需要补充】
- [配图位置1]: [配图描述]
- [配图位置2]: [配图描述]

【发布建议】
- 最佳发布时间：[建议]
- 预埋评论话题：[话题]
```

## 🔧 特殊场景处理

### 场景1：快速热点跟进
```
用户需求：紧急跟进热点，1小时内出稿

AI执行：
1. 简化学习步骤（已熟悉风格则跳过）
2. 快速抓取热点内容
3. 使用现有模板快速生成
4. 重点优化开头和标题
5. 简化检查流程（只检查关键项）
```

### 场景2：深度分析文章
```
用户需求：深度思考类文章，需要专业视角

AI执行：
1. 重点使用深度认知写作.md框架
2. 搜索专业背景资料
3. 强化历史回顾和专业分析部分
4. 增加哲学式升华内容
5. 设计开放性结尾引发讨论
```

### 场景3：工具/产品更新公告
```
用户需求：工具新版本发布文章

AI执行：
1. 学习现有更新文章风格
2. 强调迭代过程和数据（XX天、XX星）
3. 突出用户反馈驱动改进
4. 对比版本差异（配图）
5. 赋能式结尾
```

### 场景4：方法教程文章
```
用户需求：教读者使用某个工具/方法

AI执行：
1. 金句开场（引用式）
2. 揭示误区，制造冲突
3. 步骤清晰（一步一步）
4. 多案例验证（至少3个）
5. 效果对比（前后对比图）
6. 可复用性强调
```

### 场景5：不同风格适配
```
专业严谨型：
- 增加数据和引用
- 减少口语化表达
- 强化专业分析
- 适度使用专业术语

亲和力强型：
- 第一人称视角
- 情绪化表达
- 适度自嘲
- 口语化表达

幽默风趣型：
- 增加幽默元素
- 使用生动比喻
- 强化调侃和吐槽

极简干货型：
- 减少铺垫
- 精炼表达
- 强化实用性
- 删除冗余修饰
```

## 📊 质量保证机制

### 多维度检查
```
1. 人设一致性检查
   - 是否符合作者配置.md
   - 是否使用第一人称（如适用）
   - 是否有真实感
   - 是否有情绪化表达（根据风格）
   - 是否有适度自嘲（根据风格）
   - 是否赋能读者

2. 风格一致性检查
   - 与学习到的文章风格对比
   - 与作者配置.md对比
   - 段落长度是否一致
   - 开头方式是否一致
   - 专业程度是否一致

3. 爆款要素检查
   - 标题是否吸引
   - 开头是否抓人
   - 内容是否有用
   - 互动是否到位
   - 金句是否易传播

4. 实用性检查
   - 是否有可落地方法
   - 是否有具体案例
   - 是否解决了痛点
   - 是否降低了门槛
```

## 💡 协同工作机制

### AI内部协同流程
```
开始创作
    ↓
检查作者配置 ← 如果不存在，引导填写
    ↓
加载所有Skill文件 ← 必须读取所有，不能遗漏
    ↓
学习用户文章/风格 ← 至少参考2篇相似文章或配置
    ↓
抓取热点内容 ← 确保时效性
    ↓
综合分析 ← 交叉验证，确保一致性
    ↓
生成方案 ← 同时考虑多个维度
    ↓
用户确认 ← 互动迭代
    ↓
创作内容 ← 严格遵循人设和模板
    ↓
去AI化处理 ← 去除机械感
    ↓
优化检查 ← 多维度质量保证
    ↓
输出成品
```

### 关键原则
```
✅ 不单独依赖某个Skill，必须综合使用
✅ 不偏离用户已形成的写作风格
✅ 不忽略作者配置.md的人设设定
✅ 不忽略热点和时效性
✅ 不降低爆款要素标准
✅ 不遗漏人设特征
```

## 🎯 成功标准

一篇文章完成后，应该达到：

```
✅ 人设鲜明：读起来就是作者本人写的
✅ 风格统一：与现有文章风格一致
✅ 痛点精准：直击读者真实需求
✅ 方法实用：有具体可操作的步骤
✅ 要素齐全：标题、开头、内容、互动、金句
✅ 时效性强：结合热点或行业趋势
✅ 传播潜力：有易分享的金句
✅ 赋能感强：读者觉得"我也能做到"
✅ 自然流畅：无明显AI味
```

## ⚠️ 重要提示

### 对于首次使用的用户

如果检测到用户没有填写作者配置.md，AI应该：

1. **友好引导**：
```
"为了让文章更符合你的风格，我需要了解一些信息：
- 你的笔名是什么？
- 你的主要写作领域是什么？
- 你希望文章风格是（亲和力强/专业严谨/幽默风趣/极简干货）？
- 能否提供1-2篇你觉得能代表你风格的文章？"
```

2. **临时使用默认配置**：
如果用户不想填写，使用通用默认配置，但告知效果可能不够个性化。

3. **持续学习**：
随着用户创作更多文章，AI应该持续学习和调整风格。

### 风格冲突处理

当Skill通用模板与作者个人风格冲突时：

1. **作者配置优先**：以作者配置.md为准
2. **提示用户**：告知可能的风格冲突
3. **灵活调整**：在保持爆款要素的前提下适配风格
