# 去AI化润色 Skill

> 让AI生成的文本更像真人手笔，自然流畅有人味儿

## 命令说明

### /去AI化
**用法**: `/去AI化 [文章段落/全文]`

**功能**:
- 将AI生成的文本转化为自然、流畅、有节奏感的中文
- 模拟真人作者的手笔
- 增加人性化细节、场景化描述
- 打破模板化结构，让文章更有个人风格
- 替换AI高频词汇，避免机械化表达
- **根据作者配置.md调整风格**

---

## 核心润色原则

### 1. 语言风格
✅ **自然流畅有节奏感**
- 长短句交错，避免单调节奏
- 适度使用口语化表达（如"说起来"、"实际上"）
- 模拟真人思考过程的轻微冗余

❌ **避免机械化连接词**
- 不使用"首先、其次、最后"、"总之"、"综上所述"
- 避免"应该注意的是"、"值得注意的是"、"毫无疑问"、"一般来说"

### 2. 语态调整
✅ **主动语态优先**
- "我发现" 替代 "被发现"
- "这个工具让我" 替代 "我被这个工具"

❌ **被动语态改为主动**

### 3. 细节丰富度
✅ **增加人性化元素**
- 在合适位置加入个人见解
- 添加场景化描述或具象比喻（避免老套比喻）
- 使用适度冗余：限定词（"在多数情况下"、"或许"）、轻微语气词

❌ **避免完美主义**
- 可轻微调整论点，使其更像个人观点而非绝对真理
- 保留适度不确定性

### 4. 结构调整
✅ **打破模板结构**
- 避免明显的"总分总"或"并列式"
- 调整段落衔接，让逻辑自然过渡
- 随机将部分陈述句改为反问句或设问句

❌ **避免刻板格式**

### 5. 词汇选择
✅ **具体生动**
- 使用更具体的动词和名词
- 减少抽象词汇
- 替换AI高频词汇

## 使用示例

### 示例1: 理论阐述类文本

**原文**:
```
首先，人工智能技术在教育领域的应用具有重要意义。其次，它可以帮助教师
减轻工作负担。最后，它能够提供个性化的学习体验。值得注意的是，这种
技术在应用过程中也存在一些挑战。
```

**去AI化后（通用风格）**:
```
说起来，人工智能在教育里能做的事，比我们想象的要多得多。

它能帮老师分担不少重复性工作——批改作业、整理错题，这些机械的事交给
AI，老师就能把精力放在更重要的事情上。个性化学习也是一大亮点，每个
孩子都能得到量身定制的练习方案。

当然，事情没那么简单。技术落地时遇到的问题，或许比我们预想的还要多。
```

**去AI化后（专业严谨风格）**:
```
人工智能技术在教育领域的价值，主要体现在两个方面。

一是减轻教师的工作负担。批改作业、整理错题这些重复性工作，完全可以
交给AI处理，让教师把精力投入到更有价值的教学环节。二是个性化学习。
AI可以为每个学生定制练习方案，这在传统教学模式下是很难实现的。

但技术落地并非易事，实际应用中面临的挑战比想象中更多。
```

**改动说明**:
- ✅ 去掉了"首先、其次、最后"
- ✅ 增加了"说起来"等口语化表达（通用版）
- ✅ 被动改主动
- ✅ 增加了"或许"等限定词
- ✅ 长短句交错，节奏更自然

### 示例2: 工具介绍类文本

**原文**:
```
PromptFill是一款AI辅助的提示词管理工具。它支持模板功能和变量替换。
此外，该工具还具有团队协作和云端同步功能。综上所述，PromptFill能够
有效提高用户的AI创作效率。
```

**去AI化后（亲和力强风格）**:
```
用PromptFill的时候，我最大的感受是——为什么没人早点做出这种东西？

模板和变量替换这两个功能搭配起来，就像给提示词装上了乐高积木。以前
每次写提示词都要从头想，现在填几个空就能搞定。团队协作也挺有意思，
几个人可以一起维护一套提示词库，互相借鉴——这比自己闷头搞效率高多了。

效率提升？那是肯定的。但我更在意的是，它让用AI这件事变得没那么枯燥了。
```

**去AI化后（专业严谨风格）**:
```
PromptFill最核心的价值在于模板和变量替换功能。

这两个功能配合使用，可以将提示词模块化、积木化。以前每次都要从头构思
提示词，现在只需要填充变量即可。团队协作功能也很实用，多人可以共同
维护提示词库，实现知识沉淀和共享。

除了效率提升，更重要的是它改变了使用AI的方式——从重复劳动变成了
创造性工作。
```

**改动说明**:
- ✅ 开头用反问制造互动（亲和力版）
- ✅ 增加"乐高积木"的具象比喻
- ✅ 添加个人感受（"最大的感受"、"我更在意的是"）
- ✅ 去掉"综上所述"等AI高频词
- ✅ 适度口语化（"那种东西"、"闷头搞"）（亲和力版）

### 示例3: 方法教程类文本

**原文**:
```
学习ComfyUI需要掌握以下几个步骤。第一，了解节点编辑的基本概念。第二，
熟悉常用节点的功能。第三，通过实践项目提升技能。应该注意的是，学习过程
需要保持耐心和持续练习。
```

**去AI化后（亲和力强风格）**:
```
学ComfyUI这事儿，说来也简单，但刚开始确实容易懵。

节点编辑的概念你得先搞明白——就像搭积木，每个节点是一个小功能块，连
在一起就是完整的工作流。常用的节点其实就那么些，多摆弄几次也就熟了。

接下来就是动手做项目。我个人建议从简单的开始，别一上来就想搞什么大作
业，容易把自己劝退。耐心？那是必须的。但更重要的是，你得享受这个过程
的乐趣，不然很难坚持下来。
```

**去AI化后（专业严谨风格）**:
```
学习ComfyUI有三个关键步骤。

第一步是理解节点编辑的概念。可以把节点看作积木块，每个节点完成特定
功能，连接起来就形成工作流。第二步是熟悉常用节点。常用的节点数量有限，
通过几次实践就能掌握。第三步是项目实践。建议从简单项目开始，逐步提升
难度，避免因难度过高而放弃。

当然，耐心和持续练习是必要的。但更重要的是在实践中找到乐趣，这样才能
长期坚持。
```

**改动说明**:
- ✅ 去掉"第一、第二、第三"的编号
- ✅ 增加"说来也简单"、"容易懵"等口语（亲和力版）
- ✅ 使用"搭积木"的类比
- ✅ 添加个人建议和个人观点
- ✅ 将"应该注意的是"改为自然过渡
- ✅ 最后用反问增强互动

### 示例4: 观点论述类文本

**原文**:
```
毫无疑问，设计师需要掌握AI工具。一般来说，AI工具可以提高工作效率。此外，
它还能激发创作灵感。因此，设计师应该积极学习AI技术，以适应行业发展。
```

**去AI化后（亲和力强风格）**:
```
设计师要不要学AI？这个问题现在好像已经没什么讨论的必要了——答案很明确。

但怎么学，学到什么程度，这事儿倒值得琢磨。我见过太多人一股脑把所有AI
工具都装上，结果到头来真正用熟练的也就一两个。效率提升是肯定的，但更
有意思的是，AI有时候会给你一些完全没想到的灵感方向——就像有个不按
套路出牌的助手。

与其焦虑"会不会被淘汰"，不如找个顺手的工具先玩起来。反正技术这东西，
你不主动去碰，它迟早也会找上门来。
```

**去AI化后（专业严谨风格）**:
```
设计师是否需要掌握AI工具，这个问题已经没有争议——答案是肯定的。

关键在于如何学、学到什么程度。观察到一种现象：很多人安装了大量AI工具，
但真正熟练使用的只有少数几个。效率提升是显而易见的，但更重要的价值在于
AI能够提供意想不到的灵感方向——它像是一个不受常规思维限制的协作伙伴。

与其焦虑于"会不会被淘汰"，不如选择合适的工具开始实践。技术的普及是
必然趋势，主动适应才是明智之举。
```

**改动说明**:
- ✅ 开头用设问句吸引注意
- ✅ 去掉"毫无疑问"、"一般来说"
- ✅ 添加具体场景描述（装一堆工具却只用到一两个）
- ✅ 使用"不按套路出牌的助手"这种生动比喻（亲和力版）
- ✅ 去掉"因此"等连接词，用自然过渡
- ✅ 增加个人观点和态度

## 特殊场景处理

### 场景1: 保留必要逻辑
当原文有严格的逻辑顺序时，不要强行打乱，而是用自然的方式衔接:

❌ **不好**:
```
第一点，配置环境。第二点，安装依赖。第三点，运行代码。
```

✅ **更好**:
```
配置好环境之后，接下来就是安装依赖。这两步搞定，就可以运行代码了。
```

---

### 场景2: 技术术语
专业术语不需要改动，但可以在后面加通俗解释:

❌ **不好**:
```
该工具采用基于Transformer的架构，具有注意力机制。
```

✅ **更好**:
```
工具用的是Transformer架构——简单说就是给模型加了"注意力"，让它能
更好地理解上下文关系。
```

---

### 场景3: 数据引用
准确的数据和引用不要"人性化"改动，但可以加解读:

❌ **不好**:
```
用户增长了约50%，大约2000人。
```

✅ **更好**:
```
用户涨了50%——差不多2000人。这个增速说实话有点出乎我的意料。
```

## 使用建议

### 何时使用
- ✅ AI生成初稿后，进行人工化润色
- ✅ 翻译文本的中文化处理
- ✅ 技术文档的通俗化改写
- ✅ 让文章更符合个人风格

### 如何获得最佳效果
1. **提供背景**: 告诉我这篇文章的目标读者是谁
2. **说明风格**: 你希望更像资深记者、年轻博主，还是其他风格？
3. **提供范例**: 如果你有参考文章，我可以学习你的写作风格
4. **分步润色**: 长文章可以分段润色，效果更好
5. **参考作者配置**: 确保风格与作者配置.md一致

### 组合使用
```
/去AI化 + /内容优化 = 先去除AI味，再增加亲和力
/去AI化 + /爆款检查 = 确保自然流畅且符合爆款标准
/去AI化 + /风格适配 = 去AI味并匹配作者风格
```

## 风格适配示例

### 专业严谨型
```
特点：减少口语化、增强逻辑性、使用专业术语

去AI化时会：
- 减少"说实话"、"这事儿"等口语
- 增强逻辑连接
- 适度使用专业术语并解释
- 客观陈述，减少情绪化表达
```

### 亲切幽默型
```
特点：增加口语化、使用幽默比喻、情绪化表达

去AI化时会：
- 增加"说实话"、"怎么说呢"等口语
- 使用生动比喻
- 添加情绪化词汇
- 增加个人观点和吐槽
```

### 极简干货型
```
特点：精炼表达、直击要点、减少修饰

去AI化时会：
- 删除冗余表达
- 直击核心要点
- 减少形容词和修饰词
- 强化逻辑结构
```
