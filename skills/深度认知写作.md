# 深度认知写作 Skill

> **通用版** - 认知升级型内容写作方法论

## 核心写作 DNA

这套方法论的核心不是教你怎么写"爆款文章"，而是如何写出**改变读者认知的内容**。

**五个核心原则**（可根据作者调整）：

1. **身份优先于行动** - 你不是缺乏自律，而是你内心深处并不想要你嘴上说想要的东西
2. **所有行为都有目的** - 拖延不是懒，是你在保护自己免受某种恐惧
3. **改变从认知开始** - 看清问题本质，行动自然跟上
4. **反对盲目专精** - 在这个时代，多元兴趣是超能力，不是缺陷
5. **写作即思考** - 表达不清楚，说明还没想清楚

**注意**：作者可根据自己的领域和理念调整这些核心原则。

## 文章结构（六部分框架）

### 第一部分：钩子开场（前150字）

开场的唯一目标：让读者**不得不**继续读下去。

**类型A：反向认同**
```
你大概率会放弃你的新年目标。
没关系，大多数人都会。
研究显示，80-90%的新年计划会以失败告终。但问题不在于你不够努力——
而在于你从一开始就用错了方法。
```

**类型B：自我揭露**
```
三年前，我每天工作14小时，却越来越穷。
不是因为不够努力，而是我一直在用错误的方式努力。直到我想明白一件事，
一切才开始改变。
```

**类型C：反常识宣言**
```
"自律"是这个时代最大的骗局之一。
那些真正厉害的人，从来不觉得自己在"坚持"什么。他们只是没办法不那样做。
这两者的区别，决定了你能走多远。
```

**类型D：直击痛点**
```
如果你也有过这种感觉——
明明知道该做什么，就是做不到。
明明有很多想法，就是无法行动。
明明不想这样下去，却一年又一年重复。
那这篇文章是写给你的。
```

---

### 第二部分：痛点共鸣

在提供解决方案之前，先让读者感受到"你懂我"。

**核心框架**：
```
说实话，[承认自己曾经也有同样的问题]。
我试过[列举常见但无效的解决方案]。结果呢？[描述失败的结果]。
后来我才明白，问题不在于[表面原因]，而在于[深层原因]。
```

**关键点**：
- 不要上来就给答案，先共情
- 展示你经历过同样的挣扎
- 让读者觉得"不是只有我这样"

---

### 第三部分：价值承诺

告诉读者值得花时间读完这篇文章。

```
今天，我想分享[数字]个关于[主题]的核心认知。
这不是那种看完就忘的鸡汤。
这是我花了[时间]才想明白的东西。如果你愿意认真读完，至少能帮你
少走[时间]的弯路。
我们开始。
```

---

### 第四部分：核心内容（3-7个模块）

每个模块不是简单的知识点罗列，而是**认知升级的过程**。

**模块结构模板**：

```
## 一、[模块标题：点出核心洞见]

> "引用一句名言或金句"
> ——出处

[先说大多数人的错误理解]

但真相是：[给出正确的理解]

举个例子：[用具体案例说明]

*所以关键不是[表面做法]，而是[深层原则]。*
```

**写作要点**：
- 先破后立：先指出错误认知，再给出正确认知
- 用案例支撑：每个观点必须有具体例子
- 对比强烈：错误做法 vs 正确做法
- 金句点缀：每模块至少1-2句可摘抄的金句

---

### 第五部分：金句提炼

每篇文章至少要有3-5句可以单独摘出来发朋友圈的金句。

**金句特征**：
- 短小精悍（10-20字）
- 有对比或冲突
- 能引发共鸣
- 适合截图分享

**生成公式**：
```
[对比A] + [对比B] = [洞察]

"连[弱者]都能[成就]，你也一定可以"
"在[时间]，我们还在用[原始方式]？"
```

**在文中用斜体标记金句**，方便读者识别。

---

### 第六部分：赋权结尾

结尾不是总结，而是让读者**感到被赋能**，愿意采取行动。

**类型A：行动指南**
```
所以，如果你看到这里，我建议你做一件事：
[具体的、可立即执行的小行动]
不需要很久，[时间]就够了。
但这[时间]可能会改变你接下来的[更长时间]。
```

**类型B：思考问题**
```
最后，留一个问题给你：
[一个能引发深度思考的问题]
不用急着回答，让它在脑子里待一会儿。
有时候，好问题比好答案更有价值。
```

**类型C：认知升级**
```
记住：
[核心信息的简洁重述]
这不是什么高深的道理，但大多数人一辈子都不会真正理解它。
而你，现在已经知道了。
接下来，就看你怎么用了。
```

## 格式规范

### 段落规则
```
✅ 每段不超过3行
✅ 重要观点单独成段
✅ 转折处空一行
✅ 避免大段文字堆砌
```

### 格式标记
```
**粗体**：关键概念、重要词汇
*斜体*：核心金句、可摘抄的洞见
「」：特殊概念或反讽表达
——：解释或补充说明
> 引用块：名人名言、核心框架
```

### 标点节奏
```
✅ 善用句号断句，制造呼吸感
✅ 用问号引发思考
✅ 用破折号——制造停顿和强调
✅ 避免长句，保持阅读流畅
```

---

## 语气要求

### 要做到
```
✅ 像朋友聊天，不像老师讲课
✅ 敢说大多数人不愿意说的真话
✅ 承认自己的局限和失败
✅ 用具体例子支撑抽象观点
✅ 尊重读者的智商
```

### 要避免
```
❌ 居高临下的说教感
❌ 空洞的励志口号（"加油！你可以的！"）
❌ 堆砌概念不解释
❌ 过度正能量
❌ 长篇大论不分段
```

## ⚠️ 重要：减少条目式输出

**这是本 Skill 的核心原则之一**。

当你使用本 Skill 创作内容时，必须：

### ❌ 避免
```
1. 首先，你要做A
2. 然后，你要做B
3. 最后，你要做C

或者：

- 要点一：XXX
- 要点二：XXX
- 要点三：XXX
```

### ✅ 采用
```
做这件事的时候，大多数人会直接从A开始。但我的建议是，先花点时间
想清楚B，然后再回头做A。为什么？因为[原因]。至于C，那是水到渠成
的事，不用急。

整个过程的逻辑是这样的：先[理解本质]，再[采取行动]。很多人搞反了，
所以才会[常见的错误结果]。
```

**转换技巧**：

1. **用段落替代列表** - 把每个要点展开成一个小段落
2. **用连接词替代编号** - "首先→其次→最后" 改为 "开始→接下来→重要的是"
3. **用逻辑替代罗列** - 告诉读者为什么是这个顺序，而不是只列出顺序
4. **用对话替代说明** - 用"你可能会问"来引入下一点，而不是用"第二点"

**例外情况**：

以下情况可以适度使用条目式：
- 检查清单（checklist）
- 对比表格（table）
- 多个并列的简短选项（如标题建议）

但即便在这些情况下，也要：
- 控制数量（不超过5-7项）
- 每项尽量简短
- 前后用流畅的段落包裹

## 适用场景

这套方法论适合以下类型的内容：

### ✅ 适合
```
- 认知升级类文章
- 方法论总结
- 经验复盘
- 思考感悟
- 行为改变类内容
- 深度分析
```

### ⚠️ 部分适合
```
- 工具评测（可借鉴开头和结尾）
- 教程类（可借鉴痛点共鸣和案例说明）
- 技术文章（可借鉴认知升级的框架）
```

### ❌ 不太适合
```
- 纯新闻资讯
- 数据报告
- API文档
```

本 Skill 与 `公众号爆款写作.md` 是**互补关系**，不是替代关系。

```
公众号爆款写作.md     →  适合工具介绍、教程类内容
深度认知写作.md      →  适合认知升级、思考类内容

两者结合             →  既有实用价值，又有认知深度
```

**建议使用顺序**：

1. 先用 `公众号爆款写作.md` 确定人设、标题、基本结构
2. 再用本 Skill 深化内容，特别是：
   - 开头钩子设计
   - 痛点共鸣营造
   - 金句提炼
   - 赋权结尾
3. 用 `去AI化润色.md` 去除AI味，让文字更自然
4. 用 `写作辅助工具.md` 做最终检查

## 快速检查清单

文章写完后，对照这个清单自查：

```
结构检查
  □ 开头3秒抓住了注意力
  □ 有痛点共鸣，读者感受到"被理解"
  □ 核心观点清晰，不是观点堆砌
  □ 有具体案例支撑每个观点
  □ 有3-5句可传播的金句
  □ 结尾让读者感到被赋能

格式检查
  □ 段落不超过3行
  □ 没有大段文字堆砌
  □ 关键概念用粗体标记
  □ 金句用斜体标记
  □ 转折处有空行

语气检查
  □ 像朋友聊天，不像说教
  □ 有承认自己的局限
  □ 有具体例子，不空洞
  □ 不堆砌概念
  □ 不盲目励志

流畅度检查
  □ 减少了条目式输出
  □ 用段落替代列表
  □ 用逻辑连接词替代编号
  □ 阅读节奏有呼吸感
  □ 过渡自然，不生硬
```

## 示例对比

### 条目式（避免）
```
## 如何培养阅读习惯

1. 每天固定时间阅读
2. 从感兴趣的书开始
3. 减少手机使用时间
4. 找一个阅读伙伴
5. 记录阅读心得

只要做到这5点，你就能养成阅读习惯。
```

### 流畅自然式（推荐）
```
## 如何培养阅读习惯

培养阅读习惯这事儿，说难也难，说简单也简单。

最难的不是开始，而是持续。很多人兴致勃勃地买了书，看了三天就放下了。
为什么？因为他们把这当成了一项"任务"，而不是一种"享受"。

我的建议是，先别给自己定什么"每天读一小时"的目标。从每天十分钟开始，
固定在某个时间段——比如睡前、通勤路上。关键是让这件事变成生活的一
部分，而不是额外的负担。

书的选择也很重要。别一上来就啃什么大部头，从你真正感兴趣的开始。喜
欢推理就看推理，喜欢传记就看传记。阅读的乐趣在于沉浸，不在于装点门
面。

*你有没有发现，当你在做真正喜欢的事时，从来不需要"坚持"？*

最后，找一些同好。一个人读书容易放弃，一群人就不一样了。互相推荐
好书，聊聊读后感，这些都会让你更有动力。

记住，阅读不是为了完成任务，而是为了遇见更好的自己。
```

## 记住

这套方法论的核心不是教你"怎么写"，而是帮你理清"怎么想"。

**想不清楚，就写不清楚。**
**想清楚了，文字自然就流畅了。**

当你使用本 Skill 时，最重要的是：
- 保持真诚
- 尊重读者
- 用具体的例子
- 讲清楚"为什么"，而不只是"是什么"

至于"怎么做"——那是在理解"为什么"之后，水到渠成的事。
