# 公众号爆款写作 Skill

> **通用版** - 适用于各类创作者的爆款文章写作指南

## ⚠️ 核心写作原则

### 减少条目式输出，追求流畅自然

**这是最重要的写作原则**。

当使用本 Skill 创作内容时，必须优先采用流畅的段落式表达，避免过度使用列表、编号和分点陈述。

**原因**：条目式输出虽然清晰，但缺乏情感温度和阅读节奏，容易让文章显得像"说明书"而非"真诚的分享"。

**如何做**：
- ✅ 用段落叙述替代要点罗列
- ✅ 用"首先→接下来→重要的是"等连接词替代编号
- ✅ 用逻辑递进展示关系，而非并列展示
- ✅ 在必须使用列表时，控制数量（≤5项），且前后用流畅段落包裹

**例外**：检查清单、对比表格、多个并列选项可适度使用列表，但仍需保持克制。

### 2. 标题生成 ✍️

**命令**：`/标题 [文章主题/内容]`

**功能**：提供4种类型的爆款标题

**2.1 冲突对比型**
```
公式：[弱者身份] + [意外成就]
示例：
- "我都不知道自己写了什么代码，但它在Github上火了"
- "连我都学会了[技能]，你也可以轻松搞定"
- "一个[劣势]的人，如何做出[成就]"
```

**2.2 疑问引导型**
```
公式：[痛点场景] + [核心问题]
示例：
- "节点编辑与工作流，到底什么才是AI创作的流行趋势？"
- "AI工具，到底是一个均衡器，还是一个放大器？"
```

**2.3 数字+效果型**
```
公式：[数字/时间] + [结果]
示例：
- "600星背后：我的AI工具开发日记"
- "500字提示词让我心态崩了，直到我发现了XX"
- "一个周末，我做出了改变AI创作习惯的工具"
```

**2.4 否定+反转型**
```
公式：别再[错误行为]了！[新方法]来啦
示例：
- "别再用关键词试错了！MJ风格库+NanoBanana，解锁无限风格宇宙"
```

**标题自检清单**：
- [ ] 有冲突/对比/意外
- [ ] 有具体数字或场景
- [ ] 引号强调关键词
- [ ] 长度适中（15-25字）
- [ ] 让读者产生"我也有这个问题"的共鸣

---

### 3. 文章结构 📝

**命令**：`/结构 [文章类型]`

**3.1 工具介绍型**
```
【开头钩子】150字
- 具体时间/事件："这周发生了一件魔幻的事"
- 冲突感："我，一个[身份]，竟然[成就]"
- 立即抛出产品/核心观点

【痛点场景】300-400字
- 提问引入："先问大家一个问题"
- 详细描述痛苦过程（引用式内心独白、时间成本、情绪）

【解决方案】600-800字
- 产品核心理念（不要上来就介绍功能）
- 对比展示：传统方法 vs 你的方法（图片对比）
- 三点式功能介绍 + 截图

【开发/创作故事】可选 400-500字
- 分享创作过程和思路
- 与AI/工具的协作过程
- 踩坑经历

【迭代过程】300-400字
- 版本对比图（0.1 vs 0.4）
- 用户反馈引发的改进
- 数据支撑："600星""1-2天一个版本"

【价值升华】200-300字
- 金句输出："不懂技术不再是你无法落地的借口"
- 赋能式结语："连我这种零基础都能搞定，你也一定可以"

【行动号召】100字
- 链接/项目地址
- "如果感兴趣，欢迎来提建议，给个⭐️"
```

**3.2 方法教程型**
```
【金句开场】100字
- 引用式金句："一张风格参考图，胜过一千个关键词"

【问题分析】300-400字
- 揭示误区 + 实例说明

【工具/方法介绍】500-600字
- 工具1核心优势 + 截图
- 工具2核心优势 + 截图
- 强强联合的理由

【实操演示】重点 600-800字
- Step 1/2/3 + 截图
- 多案例验证

【效果分析】200-300字
- 测试结论 + 专业点评

【方法论总结】200字
- 可复用路径强调

【互动引导】100字
- "欢迎评论区返图"
```

**3.3 深度思考型**
```
【问题引入】200字
- 引用式开场 + 制造两难选择

【背景铺垫】400-500字
- 现状描述 + 承认价值 + 引入概念

【历史回溯】300字
- 专业背景 + 建立深度

【核心分析】800-1000字
- 先抑后扬 + 详细分析 + 专业洞察

【深度追问】300字
- 回归核心问题 + 哲学式升华

【留白思考】100字
- 不给标准答案，引发讨论
```

**篇幅控制**：
- 快速教程型：2000-2500字（3-5分钟）
- 深度评测型：3000-4000字（6-8分钟）
- 思考分析型：2500-3500字（5-7分钟）

**段落控制**：
- 每段不超过5行
- 每隔300字插入一张图片
- 用小标题分隔（每500-800字一个）

## 金句生成器 💡

**设计原则**：
1. 短小精悍（10-20字）
2. 有对比或冲突
3. 能引发共鸣
4. 适合截图分享

**示例金句**（可根据领域调整）：
- "一张风格参考图，胜过一千个关键词"
- "不懂技术不再是你无法落地的借口"
- "AI工具，到底是一个均衡器，还是一个放大器？"
- "我负责'想'，AI负责'写'"

**生成公式**：
- [对比A] + [对比B] = [洞察]
- "连[弱者]都能[成就]，你也一定可以"
- "在[时间]，我们还在用[原始方式]？"

## 避坑指南 ⚠️

### 标题坑
```
❌ 标题党太夸张：与内容不符
❌ 全是技术术语：不够通俗
❌ 太平淡无奇：没有点击欲
❌ 太长（超过30字）：显示不全

✅ 制造冲突但不失真
✅ 有具体场景
✅ 长度适中
```

### 内容坑
```
❌ 只罗列功能：没有使用体验
❌ 纯理论输出：缺乏案例支撑
❌ 没有个人观点：人云亦云
❌ 抬高门槛：让人觉得"太难了"

✅ 痛点+解决方案
✅ 有对比有案例
✅ 有独特洞察
✅ 降低门槛，赋能读者
```

### 节奏坑
```
❌ 开头太长：300字后才入正题
❌ 段落太长：大段文字没有图
❌ 信息过载：一篇文章塞太多内容
❌ 没有重点：读完记不住什么

✅ 开头3秒抓住注意力
✅ 每300字一张图
✅ 一篇文章一个核心观点
✅ 有金句输出
```
