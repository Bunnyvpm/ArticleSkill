# 贡献指南

感谢你有兴趣为公众号爆款写作 Skill 系统做出贡献！

## 行为准则

- 尊重所有贡献者
- 接受建设性批评
- 关注于什么是最适合的社区

## 如何贡献

### 报告 Bug

如果你发现了 bug，请：

1. 检查是否已有相同问题的 Issue
2. 使用清晰的问题标题
3. 提供详细的复现步骤
4. 附上相关的错误信息和截图

### 提出新功能

1. 先在 Issue 中讨论你的想法
2. 说明新功能的用例和价值
3. 提供可能的实现方案

### 提交代码

1. Fork 本仓库
2. 创建你的特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交你的更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 开启一个 Pull Request

### 改进文档

文档改进同样受欢迎！你可以：

- 修正错别字
- 改进说明的清晰度
- 添加更多示例
- 翻译文档

## 写作风格改进

如果你希望改进写作方法论，请确保：

1. **保持通用性**：改进应适用于不同领域的创作者
2. **提供示例**：包含具体的使用示例
3. **保持一致性**：与其他技能文件保持风格一致
4. **测试验证**：在实际写作中验证改进效果

## 技能文件结构

系统包含以下核心技能文件：

- `公众号爆款写作.md` - 核心写作方法论
- `深度认知写作.md` - 认知升级型内容框架
- `热点分析工具.md` - 热点话题分析
- `写作辅助工具.md` - 实用工具和命令
- `去AI化润色.md` - 文本润色规则
- `快速参考.md` - 快速查询卡片
- `文章创作工作流程.md` - AI 工作流程

修改这些文件时，请确保：

- 保持 Markdown 格式正确
- 使用清晰的标题层级
- 提供具体的示例
- 标注修改的版本和日期

## 代码规范

虽然本项目主要是文档，但仍有一些规范：

### Markdown 规范

- 使用中文标点
- 标题层级清晰（最多到三级标题）
- 代码块使用正确的语法高亮
- 表格格式整齐

### 文件命名

- 使用中文文件名（因为是中文项目）
- 文件名清晰表达内容
- 示例文件使用 `.md` 扩展名

## 测试你的改进

在提交 Pull Request 之前：

1. **在 Claude Code 中测试**：使用修改后的技能文件创作一篇文章
2. **检查输出质量**：确保改进达到了预期效果
3. **验证兼容性**：确保与其他技能文件配合良好

## 版本控制

- 在技能文件的末尾注明版本和更新日期
- 重大更新需要更新 README 中的版本号
- 保持向后兼容性

## 文档变更

如果你修改了：

- **写作方法论**：更新 `公众号爆款写作.md`
- **工作流程**：更新 `文章创作工作流程.md`
- **命令列表**：更新 `写作辅助工具.md` 和 `快速参考.md`
- **核心原则**：更新 `CLAUDE.md`

## 许可证

通过贡献，你同意你的贡献将在 MIT 许可证下发布。

## 问题和支持

如果你有任何问题，欢迎：

- 提交 Issue
- 在讨论区发起讨论
- 联系项目维护者

感谢你的贡献！🎉
